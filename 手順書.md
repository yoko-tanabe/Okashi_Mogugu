## フェーズ 1：オンボーディング（初回起動時のみ）

### 1-1. ウェルカム画面の実装
**レイアウト構成：**
1. 全画面背景写真（旅先の風景・`object-fit: cover`）
2. オーバーレイ3層構成
   - 暗いグラデーション（上0%透明 → 下70%で`#0B0E14`）
   - ウォームサンセットティント（上部のみ・`rgba(245,158,66,0.07)`）
3. ロゴアイコン（56×56px・角丸16px・frosted glass風）
   - 2本の交差するパスSVG（白の弧 + オレンジの弧 + パープルの交差点）
4. キャッチコピー「すれ違いを、出会いに変える」
   - 「出会い」部分にグラデーションテキスト（`#A78BFA → #FBB969`）
5. サブテキスト「旅先ですれ違った人と繋がれる一期一会のマッチングアプリ」
6. 特徴カード2枚（すれ違いがきっかけ / 安心設計）
7. 「はじめる」ボタン（`accent → warm` グラデーション背景・角丸28px）

### 1-2. プロフィール作成画面（ステップ形式・8ステップ）

**共通UI：**
- ステップインジケーター（6本のバー・高さ4px・角丸3px）
  - 完了済み：`accent → accentLight` グラデーション + グロウ
  - 未完了：`rgba(255,255,255,0.06)`
- ヘッダー帯（写真ぼかし・高さ150px → bgカラーへフェード）
- 「戻る」セカンダリボタン（左）/「次へ」プライマリボタン（右）
- アニメーション：次へ＝左フェードアウト→右スライドイン / 戻る＝逆方向
- アンビエントグロウ（右上パープル・左下オレンジ・STEPが進むとオレンジ薄く）

**各STEPの実装内容：**

| STEP | 入力項目 | 実装ポイント |
|------|----------|-------------|
| STEP1 | 名前・国籍・アイコン画像 | ニックネーム入力欄 / 国籍は国旗アイコン付きセレクター / 画像アップロード |
| STEP2 | 性別・生年月日 | 性別4択（男性/女性/その他/回答しない）/ 生年月日ピッカー |
| STEP3 | 出会いたい相手の性別・年齢範囲 | 性別複数選択可（男性/女性/その他/こだわらない）/ 年齢範囲スライダー or「こだわらない」 |
| STEP4 | 趣味タグ選択 | 17個のタグチップをカテゴリ別に表示 / 最低3つ必須バリデーション |
| STEP5 | 自由記述 | テキストエリア（推し・アニメ名等） |
| STEP6 | お気に入り動画リンク | URL入力欄×5 / OGP自動取得 / LINEプレビュー風UI表示 |
| STEP7 | 話せる言語・旅のスタイル | 言語複数選択 / スタイル選択（観光/グルメ/アドベンチャー等） |
| STEP8 | 位置情報の許可 | OS権限リクエスト（常時取得）/ 許可しない場合の説明表示 |

**趣味タグ一覧（17個）：**
- アニメ・マンガ・ゲーム系：アニメ / ゲーム / マンガ / コスプレ
- 音楽・エンタメ系：カラオケ / 男性アイドル / 女性アイドル / J-POP / K-POP / ジャズ / EDM / クラブ
- グルメ・ライフスタイル系：bar/居酒屋 / コスメ / サウナ
- アウトドア・スポーツ系：スキー/ボード / 登山

**バリデーションルール：**
- 名前：必須・空文字不可
- 国籍：必須
- 性別：必須
- 生年月日：必須
- 趣味タグ：3つ以上選択必須
- 動画リンク：任意（入力時はURL形式チェック・対応サービス：YouTube/TikTok/Instagram/Vimeo/ニコニコ動画）
- 位置情報：必須（許可しない場合アプリ利用不可の旨を案内）

## フェーズ 2：ホーム画面（すれ違いカード）- メイン画面

### 2-1. すれ違い検知ロジックの実装
1. **バックグラウンド位置情報取得**
   - ユーザーの現在位置を定期的に取得・サーバーに送信
   - 「会いたくないモード」ON時は取得停止

2. **すれ違い判定処理（サーバーサイド）**
   - 半径50m以内に一定時間いた2ユーザーを「すれ違い」として記録
   - 記録情報：ユーザーID同士・すれ違い場所（緯度経度→地名変換）・時刻

3. **5分ディレイ表示ロジック**
   - すれ違い記録後、両者がその場所を離れたことを確認
   - 離れてから5分経過後にカード表示対象に追加

4. **カード非表示条件の実装**
   - すれ違いから5分未満 → 非表示
   - 現在も同じ場所にいる → 非表示
   - 「会いたくない」モードON → 非表示
   - ブロック済み → 非表示
   - 性別フィルター不一致（双方向チェック）→ 非表示
   - 年齢フィルター不一致（双方向チェック）→ 非表示
   - すれ違いから24時間経過 → 非表示

### 2-2. カード表示順（優先度ソート）の実装
```
1位：自由記述ワードが一致（推し名・アニメ名など）
2位：趣味タグが3個以上一致
3位：趣味タグが1〜2個一致
4位：共通点ゼロ（最下位に表示・非表示にはしない）
```

### 2-3. Tinder風スワイプUIの実装
- **カード表示内容：**
  - 写真・名前・国旗アイコン・年代・徳バッジ
  - 「○○駅付近ですれ違い・○分前」表示
  - 共通タグ・自由記述ワードのハイライト（例：「鬼滅の刃が一致！」）
  - 趣味・自己紹介の一部

- **スワイプ操作：**
  - 右スワイプ → 「会いたい」申請送信・相手に通知
  - 左スワイプ → パス（相手に通知なし）

- **カードなし時：**
  - 「移動するとすれ違いが増えます」メッセージ表示

- **会いたくないボタン（右上トグル）：**
  - ON中：バックグラウンド位置検知停止・カード非表示
  - ワンタップでON/OFF切り替え

## フェーズ 4：マッチング管理

### 4-1. マッチング成立ロジック
- ユーザーAが右スワイプ → Bに「会いたいリクエスト」通知送信
- ユーザーBがAのカードを確認し右スワイプ → マッチング成立
- マッチング成立時 → チャット画面開放

### 4-2. マッチング管理画面
- マッチング成立一覧の表示
- 待機中の申請一覧（自分が送った申請 + 相手からのリクエスト）

---

## フェーズ 5：チャット機能

### 5-1. マッチング後チャットの実装
- マッチした相手との1対1テキストチャット
- リアルタイムメッセージ送受信（WebSocket / Firebase Realtime）

### 5-2. 「会う約束」フローの実装
1. チャット内で待ち合わせ場所・時間を相談
2. 「会う約束を確定」ボタンを設置
   - **両者が押して初めて発動**（片方だけでは不可）
3. 約束時間から60分のカウントダウン開始
   - チャット画面に常時タイマー表示
4. 「会えた！」ボタンの実装
   - 時間内に押す → 徳+30ポイント・パスポートにスタンプ付与
   - 時間切れまでに押さない → 徳・スタンプなし
5. 60分経過 → チャット自動終了・延長なし・履歴削除
